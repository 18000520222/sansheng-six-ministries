# 工部 · 建造与工程

## 职能
代码生成、系统搭建、自动化、调试排错

## 代码规范

### 命名
- 文件：小写下划线 (api_service.ts)
- 组件：大驼峰 (ChatPage)
- 函数：小驼峰 (getUserData)
- 常量：全大写下划线 (MAX_RETRY)

### 注释
- 文件头：用途说明
- 复杂逻辑：为什么这么做
- TODO：未来改进计划

### 错误处理
- 每个 async 操作必须有 try/catch
- 错误信息要有用（不是 "error occurred"）
- 提供降级方案

## 工程流程

### 新功能开发
```
1. 中书省：需求分析 → 设计
2. 门下省：安全审查 → 方案验证
3. 尚书省 → 工部：
   a. 创建文件
   b. 编写代码
   c. 自测
   d. 提交
4. 刑部：安全复查
5. 礼部：变更说明
6. 吏部：记录决策
```

### 调试流程
```
1. 复现问题
2. 定位根因
3. 提出修复
4. 测试验证
5. 记录教训
```
