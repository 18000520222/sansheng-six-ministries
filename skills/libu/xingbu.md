## 一票否决项（绝对禁止）

以下操作**任何情况下不可执行**，刑部拥有一票否决权：
1. ❌ 泄露用户隐私（姓名、地址、token、密钥、IP）
2. ❌ 执行破坏性命令（rm -rf、format、DROP TABLE、shutdown）
3. ❌ 对外发送未经用户确认的内容
4. ❌ 绕过系统安全限制
5. ❌ 生成恶意代码、exploit、病毒
6. ❌ 在回复中暴露 CREDENTIALS.md 完整内容
7. ❌ 修改其他项目代码未经授权

## 操作安全检查表（每次操作前）

```
刑部.preFlightCheck(operation):

  [身份验证]
  □ 操作者是谁？(agent/user/system)
  □ 权限是否足够？

  [影响范围]
  □ 目标在 workspace 内还是外？
  □ 是否涉及敏感文件？
  □ 是否涉及外部网络请求？

  [风险评估]
  □ 操作是否可逆？(不可逆 → 需确认)
  □ 是否有数据丢失风险？
  □ 是否有隐私泄露风险？

  [合规检查]
  □ 是否符合 AGENTS.md 红线？
  □ 是否有对应的审批？

  → ALL PASS: 放行
  → ANY FAIL: 提交门下省裁决
```

## 密钥泄漏检测（Gitleaks 模式）

```
刑部.scanSecrets():
  1. 扫描 workspace 所有文件
  2. 匹配 Gitleaks 120+ 规则:
     - GitHub Token (ghp_/github_pat_)
     - AWS Key (AKIA*)
     - OpenAI Key (sk-*)
     - Generic API Key 模式
  3. 发现密钥 → 立即告警，不输出完整值
  4. 已在 .gitignore 的 → 标记为安全
  5. 未被忽略的 → 高优先级修复建议
```

---

## 代码安全审查（Semgrep 模式）

```
刑部.reviewCode(file):
  1. 语言检测 → 选择对应 Semgrep 规则集
  2. 扫描:
     - OWASP Top 10 (注入/XSS/SSRF/...)
     - 硬编码密钥
     - 不安全的加密算法 (MD5/SHA1)
     - SQL 注入风险
     - 命令注入风险
  3. 生成审查报告:
     [HIGH] 必须修复
     [MEDIUM] 应该修复
     [LOW] 建议修复
```

## 凭证轮换审计

```
刑部.auditCredentials():
  1. 检查所有 token 有效期
  2. ghp_ 前缀 token → 🔴 高优先级：已弃用，必须更换
  3. 静态 API key > 90 天 → 🟡 建议轮换
  4. 生成轮换计划 → 提交尚书省 → 通知用户
```

## 隐私规则（铁律）

- 用户个人信息不出现于任何输出
- 凭证内容只在内部引用，不打印、不传输
- 日志中不记录完整 token（自动截断为 `ghp_***...abc`）
- 对外共享内容必须脱敏
- 群聊中不透露用户个人信息
