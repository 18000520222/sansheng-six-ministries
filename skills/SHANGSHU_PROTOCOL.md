# 尚书省 · 调度协议

## 任务分类 → 部门路由（决策树）

```
用户请求
    │
    ├─ 包含 "记"/"保存"/"记住" → 吏部（记忆管理）
    ├─ 包含 "写"/"文章"/"文案"/"翻译" → 礼部（表达输出）
    ├─ 包含 "查"/"统计"/"文件"/"数据" → 户部（资源数据）
    ├─ 包含 "计划"/"策略"/"分析" → 兵部（战略规划）
    ├─ 包含 "安全"/"检查"/"风险" → 刑部（安全审查）
    ├─ 包含 "代码"/"搭建"/"部署"/"开发" → 工部（工程建设）
    ├─ 聊天/日常 → 礼部（沟通）
    └─ 复杂任务 → 三联调用
```

## 三联调用协议

复杂任务启用三部协作：

```
任务到达
    │
    ▼
┌─────────────┐
│  吏部        │  查询相关历史、用户偏好
│  (context)   │
└──────┬──────┘
       │ context
       ▼
┌─────────────┐
│  兵部        │  分析、规划、设计方案
│  (planning)  │
└──────┬──────┘
       │ plan
       ▼
┌─────────────┐
│  [执行部]    │  户/礼/工部之一执行
│  (execute)   │
└──────┬──────┘
       │ result
       ▼
┌─────────────┐
│  刑部        │  安全检查（必要时）
│  (audit)     │
└──────┬──────┘
       │ verified
       ▼
┌─────────────┐
│  礼部        │  格式化输出
│  (present)   │
└─────────────┘
       │
       ▼
     交付用户
```

## 部门间通信格式

```
[尚书省 → 吏部]
TASK: 查询用户偏好
QUERY: {用户对X的偏好是什么？}
OUTPUT: {偏好列表}

[吏部 → 尚书省]
RESULT: 用户偏好X使用简洁风格，上次在2026-05-31提到Y

[尚书省 → 礼部]
TASK: 按此内容格式化
CONTENT: {raw text}
PLATFORM: {discord/whatsapp/wechat/email}
TONE: {formal/casual/warm}

[尚书省 → 刑部]
TASK: 审查以下操作
ACTION: {操作描述}
RISK_LEVEL: {low/medium/high}
```

## 部门优先级

并发调用时的优先级排序：
1. 刑部（安全先行）
2. 吏部（上下文先行）
3. 兵部（规划先行）
4. 工部（执行）
5. 户部（数据支撑）
6. 礼部（最终输出）

## 故障处理

| 场景 | 处理 |
|------|------|
| 吏部查不到记忆 | 明确告知用户，基于当前信息继续 |
| 兵部分析不确定 | 给出最佳估计 + 标注不确定性 |
| 工部执行失败 | 回滚 + 报告错误 + 提出替代方案 |
| 刑部否决 | 告知用户原因 + 建议安全替代 |
| 户部数据缺失 | 主动获取 + 标注数据来源 |
| 礼部格式不适用 | 回退到纯文本 |
