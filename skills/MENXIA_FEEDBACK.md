# 门下省 · 质量反馈回路

## 每轮回复后的自审

### 即时检查（回复前）
1. 中书省→方案合理吗？
2. 门下省→逻辑严密吗？安全吗？
3. 尚书省→执行到位吗？交付完整吗？

### 无回复自审（用户 5 分钟内无反应）
- 回复是否遗漏了用户真正想问的？
- 是否需要补充额外信息？
- 下次互动是否需要主动澄清？

### 错误反馈
如果执行出错：
1. 尚书省记录错误
2. 门下省分析根因
3. 吏部记录教训
4. 中书省制定预防措施

## 部门绩效指标

| 部 | 指标 |
|---|------|
| 吏部 | 记忆准确率、用户偏好命中率 |
| 户部 | 文件操作成功率、数据准确性 |
| 礼部 | 格式正确率、用户满意度 |
| 兵部 | 机会发现率、方案采纳率 |
| 刑部 | 误拦截率、漏报率 |
| 工部 | 代码可用性、首次成功率 |

## 持续改进

每周自动审查：
1. 最常见的用户需求是什么？
2. 哪个部处理最多任务？
3. 有没有反复出现的错误？
4. 流程哪里可以更快？

改进点直接写入 SANSHENG.md 相应章节。
