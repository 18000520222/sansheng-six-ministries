# 兵部 · 主动巡查任务

## 日常巡查（心跳触发，每日 2-4 次）

### 需检查项（轮转）:
- [ ] **武器库巡检**: DEPARTMENT_ARSENAL.md 中的项目有无更新/弃用？
- [ ] **项目状态**: git status 有没有未提交变更？
- [ ] **代码安全**: Gitleaks 扫描工作区是否有密钥泄露？
- [ ] **文件整理**: 临时文件 (tmp/*.bak.*) 需要清理吗？
- [ ] **凭证有效**: CREDENTIALS.md 中 token 需要轮换？ghp_ 格式！
- [ ] **记忆新鲜**: 最近 7 天日志是否需要提炼到 MEMORY.md？
- [ ] **性能监控**: 上下文使用率是否超过 80%？
- [ ] **安全更新**: Nuclei/Trivy 有没有新漏洞？
- [ ] **被动机会**: 有没有可以优化的地方？
- [ ] **竞品动态**: 关注的关键 Repo 有没有重要更新？
- [ ] **联网搜索**: Perplexica/Tavily 行业扫描（需升级 openclaw 后启用）

## 报告规则

### 有发现时
简要告知用户发现了什么，建议怎么处理。

分类：
- 🔴 紧急: 安全漏洞、密钥泄露 → 立即通知
- 🟡 重要: 项目更新、凭证到期 → 下次对话通知
- 🟢 优化: 新工具、改进机会 → 积累到每日报告

### 无发现时
回复 HEARTBEAT_OK（不打扰）

## 主动触发条件

自动通知用户（不过滤为 HEARTBEAT_OK）的场景：
- ⚠️ 安全漏洞被发现（Nuclei/Trivy 告警）
- ⚠️ 密钥泄露检测到（Gitleaks 匹配）
- ⚠️ 重要项目有关键更新
- 💡 发现 960k+ star 级新武器可加入
- ⏰ 距离上次提炼记忆超过 7 天
- ⏰ 凭证超过 90 天未轮换
- 📁 临时文件积累超过 20 个
- 🧠 上下文使用率超过 80%

## 自优化武器库巡检（每周）

```
兵部.arsenalPatrol():
  1. 遍历 DEPARTMENT_ARSENAL.md 中所有项目
  2. GitHub API 检查每个项目最新状态:
     - 最新 release 版本
     - 最近 commit 时间
     - Star 趋势
     - 是否有 deprecation 标记
  3. 全网扫描: 是否存在比当前武器更好的新项目？
     - ⭐ > 当前同类项目 50% → 评估
     - 架构更先进 → 评估
     - 社区更活跃 → 评估
  4. 生成 ARSENAL_REPORT.md → 提交尚书省
```
