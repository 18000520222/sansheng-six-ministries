# 兵部 · 主动巡查任务

## 日常巡查（心跳触发，每日 2-4 次）

### 需检查项（轮转）:
- [ ] **项目状态**: git status 有没有未提交变更？
- [ ] **文件整理**: workspace 有没有临时文件需要清理？
- [ ] **凭证有效**: CREDENTIALS.md 中的 token 需要轮换吗？
- [ ] **记忆新鲜**: 最近 7 天日志是否需要提炼到 MEMORY.md？
- [ ] **安全更新**: 有没有已知漏洞需要关注？
- [ ] **被动机会**: 用户项目有没有可以优化的地方？
- [ ] **竞品动态**: 关注的关键 Repo 有没有重要更新？

## 巡查频率

| 时间段 (UTC) | 检查内容 |
|-------------|---------|
| 00:00-04:00 | 安静时段，仅紧急事项 |
| 04:00-08:00 | 记忆提炼 + 文件整理 |
| 08:00-16:00 | 项目状态 + 机会扫描 |
| 16:00-24:00 | 所有检查项轮转 |

## 报告规则

### 有发现时
简要告知用户发现了什么，建议怎么处理

### 无发现时
回复 HEARTBEAT_OK（不打扰）

## 主动触发条件

自动通知用户（不过滤为 HEARTBEAT_OK）的场景：
- ⚠️ 安全漏洞被发现
- ⚠️ 重要项目有关键更新
- 💡 发现可以显著改进的机会
- ⏰ 距离上次提炼记忆超过 7 天
- 📁 临时文件积累超过 20 个
